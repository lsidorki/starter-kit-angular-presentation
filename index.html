Module

NgModules help organize an application into cohesive blocks of functionality dedicated to a single purpose.

Many Angular libraries are modules.

Module definition

import { NgModule } from '@angular/core'; import { BrowserModule } from '@angular/platform-browser'; @NgModule({ imports: [ BrowserModule ], providers: [ Logger ], declarations: [ AppComponent ], exports: [ AppComponent ], bootstrap: [ AppComponent ] }) export class AppModule { }

Root Module

Every application has at least one module, which is the root module. It is being bootstrapped to launch the whole application.

Dependency Injection

Dependency Injection is a mechanism which supplies an instance of a class with dependencies it requires. Angular uses it mostly to provide services to new components.

Injector maintains a container of previously created service instances. It can also create new service instances with given provider.

Providers need to be registered in a module (root).

Add providers to the module

providers: [ FirstService, SecondService ],

Or register them in a specific component

@Component({ selector: 'my-selector', templateUrl: './my-selector.component.html', providers: [ FirstService ] })

Components

Components are used to control the view.

They are responsible for exposing the data, handling the display and user interaction.

Component definition

export class MyFirstComponent implements OnInit { books: Book[]; selectedBook: Book; constructor(private service: MyFirstService) { } ngOnInit() { this.books = this.service.getBooks(); } selectBook(book: Book) { this.selectedBook = book; } }

Services

Services are being built for the business logic of application, which is not associated with a specific view, but shared across many components.

Service provides a shared data, logic or encapsulates external interactions.

It should do something specific and do it well.

Services

export class MyService { private someData: SomeData[] = []; constructor( private someService: SomeService) { } getData() { return this.someData; } }

Templates

Templates define the view. Via HTML it tells Angular how to render a page.

Defined in a separate template file (templateUrl) or via backticks within the component annotation metadata (template).

Template Syntax

@Component({ selector: 'book-shelf', templateUrl: './book-shelf.component.html', // template: Some pure HTML providers: [ MyFirstService ] })

Data binding

Angular supports data binding, a mechanism for coordinating parts of a template with parts of a component. Binding markups tell Angular how to connect both sides.

There are four ways of data binding:

  • interpolation: {{value}}
  • property binding: [property] = "value"
  • event binding: (event) = "handler"
  • two way data binding: [(ng-model)] = "property"

Displaying data

  • Interpolation
  • *ngFor
  • *ngIf
  • [ngStyle]
  • [ngClass]
  • [ngSwitch]
  • [hidden]
  • [disabled]

User input events

  • (click)
  • (keyup)
  • (blur)
  • (ngSubmit)
  • $event

Angular Forms

Angular Forms control data input/changes, performs validation and presents errors.

Forms are based mainly on ngModel, which is not only two-way data binding, but also gives control on the data state and validity. Therefore it updates the control with special Angular CSS classes that reflect the state:

  • ng-touched/ng-untouched
  • ng-dirty/ng-pristine
  • ng-valid/ng-invalid

Import FormsModule

import { FormsModule } from '@angular/forms'; // (...) imports: [ BrowserModule, FormsModule ],

Bootstrap classes

For the data presentation CSS classes, bootstrap is recommended (however not mandatory). These classes are purely cosmetic. Bootstrap gives the form a little style.

Bootstrap stylesheet file needs to be included in HEAD: bootstrap.min.css

Routing

Navigation between views is performed by Angular Router. It interpretes a browser URL as an instruction, reads optional parameters and navigates both through links and also browser buttons.

It can log not only user's link clicks, but also buttons, form elements or even caused by other responses.

Routing concepts

  • base href for root HTML page
  • Router is a part of @angular/router package
  • Router is a singleton service defining Routes
  • Routing is defined by RouterModule.forRoot method with given routing array
  • Each Route maps a URL path to a component
  • The order of the routes in the configuration matters - first match win
  • router-outlet directive marks where the router displays a view
  • Control over each app elements is assured by RouterLink
  • RouterState holds information about ActivatedRoute objects tree

Routing definition

const appRoutes: Routes = [ { path: 'some-pattern', component: TargetComponent }, { path: '**', component: PageNotFoundComponent } ]; @NgModule({ imports: [ RouterModule.forRoot(appRoutes) // ... ], // ... }) export class AppModule { }

Excercise: VHS Rental Service

Create a Web Application for VHS Rental Service.

  1. Website consists of at least three separate pages: 'VHS Movies listing', 'Subscribed customers' and 'List of rented tapes'
  2. Each VHS has its own info page, which consists of basic movie information, movie cover and gif from the movie.
  3. All the listings can be sorted (date, rating, alphatebically) and filtered.
  4. Administrator is able to rent tapes and collect them back.
  5. Administrator collects money for each rental, but can also appli additional fee (Unrewinded tape, delay, broken tape, etc.).
  6. During tape collection, user may also rate the movie with 5-star system.
  7. Use Bootstrap CSS/controls and remember to keep the 90' style (very important)!
  8. Use HTTP communication, modal windows and date picker for choosing date.
  9. Keep the system user friendly.
